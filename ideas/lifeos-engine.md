# LIFEOS ENGINE

## Core Principle

LifeOS is not a note-taking system.

LifeOS is a system for converting a person's identity, values, goals, knowledge, and circumstances into high-quality decisions and consistent action.

The framework is universal. The context is personal.

---

# 1. PERSON
The root object. Everything derives from the person. A person contains: Identity, Values, Vision, Domains, Goals, Systems, Playbooks, Knowledge, Metrics, Projects, Constraints.

# 2. IDENTITY
Who am I? Identity defines behavior. Examples: Parent, Student, Engineer, Founder, Athlete, Artist, Researcher, Leader. Identity is not a goal — it is a role or state of being.

# 3. VALUES
What matters most? Values establish priorities and tradeoffs. Examples: Family, Faith, Freedom, Learning, Wealth, Health, Service, Creativity. Values must be ranked; when conflicts occur, higher values win.

# 4. VISION
What does a successful life look like? Vision defines long-term direction. Horizons: 3 years, 5 years, 10 years, lifetime. Vision describes outcomes, not tasks.

# 5. DOMAINS
Life is divided into domains. Default: Spiritual, Health, Relationships, Family, Career, Education, Finance, Projects, Community, Recreation. Domains can be added, removed, or renamed.

# 6. GOALS
Goals exist within domains. Every goal contains: description, deadline, importance, expected impact, current status, success metric. Examples: get promoted, lose 10kg, publish a paper, build a business, improve relationships. Goals define destinations.

# 7. SYSTEMS
Goals require systems. A system is a repeatable process that produces an outcome. (Lose weight → nutrition, exercise, sleep, tracking. Get a job → applications, networking, skill development, interview prep.) Goals without systems are wishes.

# 8. PLAYBOOKS
Reusable operating procedures. Each contains: objective, process, metrics, templates, common mistakes, best practices. Examples: Exam Prep, Internship, Startup Validation, Leadership, Fitness. Playbooks turn knowledge into execution.

# 9. KNOWLEDGE
Reference material: books, articles, courses, frameworks, notes, videos, research. Knowledge is not action — it must be transformed: Knowledge → Principle → Playbook → Action.

# 10. PRINCIPLES
Distilled truths from knowledge and experience. Format — Principle: short statement. Why: reason it matters. When: where it applies. Examples: outcomes matter more than appearances; distribution before optimization; measure performance, not effort; systems outperform motivation.

# 11. PROJECTS
Temporary efforts with defined outcomes. Each contains: objective, scope, timeline, milestones, risks, deliverables, metrics. Projects exist to advance goals.

# 12. CONSTRAINTS
Reality matters: time, energy, money, health, responsibilities, deadlines. The system must optimize within constraints.

# 13. METRICS
What gets measured gets improved. Outcome-focused. (Health: weight, strength, VO₂ max. Career: interviews, offers, income. Education: exam scores, recall accuracy. Projects: users, revenue, completion rate.) Metrics create feedback.

# 14. DECISION ENGINE
Every decision is evaluated by: 1) alignment with values, 2) alignment with vision, 3) impact on goals, 4) return on time invested, 5) urgency, 6) reversibility, 7) risk, 8) available resources. Score actions; prioritize highest-leverage.

# 15. DAILY ENGINE
Input: goals, systems, calendar, deadlines, constraints, metrics. Output: top priorities, critical tasks, focus blocks, review points. Purpose: determine what should be done today.

# 16. WEEKLY REVIEW
What worked? failed? changed? was learned? should stop / continue / start? Update goals, systems, projects, priorities.

# 17. MONTHLY REVIEW
Evaluate domain health, goal progress, system effectiveness, resource allocation, long-term alignment. Make strategic adjustments.

# 18. CONTINUOUS IMPROVEMENT LOOP
Capture → Organize → Distill → Decide → Execute → Measure → Reflect → Improve. Repeat indefinitely.

---

# AI LAYER
The AI has access to: identity, values, vision, domains, goals, systems, playbooks, knowledge, projects, constraints, metrics, calendar. Its purpose is not generic advice — it is a reasoning engine that helps the person make better decisions, execute consistently, and continuously improve.

---

# FINAL EQUATION
Person + Context + Goals + Systems + Knowledge + Feedback = Personal Operating System.
